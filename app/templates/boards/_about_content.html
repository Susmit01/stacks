Built with Claude Code

More than a project management tool, Stacks is a demonstration of the power of Claude Code - showing how a good quality web application can be built purely on the basis of instructions, with zero manual coding.

{# ── What is Stacks + quick stats ─────────────────────────────────────────── #}

The product

What is Stacks?

Stacks is a full-stack project management platform inspired by tools like Monday.com - built in a single weekend without writing a single line of code by hand. Every route, every template, every database migration was produced by describing what was needed in plain English to Claude Code.

It lets teams organise work into boards, groups, and items - with custom columns, drag-and-drop reordering, a live search bar, and a dashboard that summarises progress across all boards at a glance.

{# ── What this wasn't ──────────────────────────────────────────────────────── #}

Honest caveats

What this wasn't

This is a portfolio project, not production software. The codebase hasn't been security-audited. It runs on free-tier infrastructure. Some features - notifications, real-time collaboration, file uploads - are deliberately out of scope.

The point was to learn how far AI-assisted development can go on a focused build, not to compete with Monday.com. Conscious trade-offs were made at every step, and that's exactly what makes this a useful experiment rather than a shipping product.

{# ── What I learned ────────────────────────────────────────────────────────── #}

Reflection

What I learned

The most surprising thing wasn't how much Claude could build - it was learning when not to delegate. Short, specific prompts consistently outperformed long, detailed ones. Polish passes mattered more than I expected: a working feature and a good feature are two very different things, and the gap between them requires human judgement, not more prompting. The hardest skill to develop was knowing where to draw the line between Claude deciding and me deciding - and realising that line moves depending on whether you're making a product choice or an implementation choice. That distinction turned out to be the real lesson.
